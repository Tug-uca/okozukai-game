# おこづかいゲーム - ゲーム企画書

## 1. ゲーム概要

### 1.1 ゲームタイトル
**「おこづかいチャレンジ！」**（仮称）

### 1.2 ターゲット
小学1年生（6〜7歳）

### 1.3 学習目標
**数字を使って計画を立てる力を育てる**

子どもたちは、このゲームを通じて以下を学びます：
- 目標を達成するために必要な金額と期間を計算する
- 毎月の支出を計画し、計画通りに実行する難しさを体験する
- 無理な計画と実現可能な計画の違いを理解する
- 計画の大きさと実効性のバランスを考える力

### 1.4 ゲームの特徴
- **作戦立案フェーズ**：目標のおもちゃを選び、毎月何を買うかを計画する
- **実行フェーズ**：お出かけの時に誘惑が発生し、計画を守るか使うかを選択する
- **フィードバック**：結果から学び、何度でもやり直せる

### 1.5 ホスティング環境
GitHub Pages（静的サイト）
- サーバーサイド処理なし
- データベースなし
- すべてブラウザ内で完結

## 2. ゲームルール

### 2.1 基本的な流れ

```
【作戦立案】→【実行（月ごとの進行）】→【結果発表】→【やり直し可能】
```

### 2.2 作戦立案フェーズ

#### 2.2.1 初期設定
- プレイヤーは毎月もらえるおこづかい額が決まっている（例：500円/月）
- 現在の所持金：0円からスタート

#### 2.2.2 目標おもちゃの選択
- ゲーム内で用意された複数のおもちゃから1つを選択する
- 各おもちゃには価格が設定されている
  - 例：ミニカー（1,500円）、ぬいぐるみ（2,500円）、ゲーム（4,000円）など

#### 2.2.3 毎月買うものを決める
プレイヤーは、「毎月買うもの」を選択します。この計画は毎月繰り返されます。

**選択肢の例：**
- 「なにも買わない」（0円）
- 様々なアイテムから選択（例：チョコレート100円、アイス150円、ミニカー300円、ぬりえ400円、レースゲーム100円、UFOキャッチャー200円など）
- すべてのアイテムは画像付きカードで表示される

**複数選択可能：**
- 毎月複数のアイテムを買う計画も立てられる
- 例：「毎月チョコレート（100円）とミニカー（300円）を買う」→ 毎月400円支出

**必要月数の自動計算：**
- 目標おもちゃの価格と毎月の支出計画から、必要な月数が自動的に計算される
- 例：目標3,000円、毎月の支出400円 → 毎月100円貯金 → 30か月必要
- 例：目標3,000円、何も買わない → 毎月500円貯金 → 6か月必要

#### 2.2.4 計画の確認
- 計画を立て終わったら、以下が表示される：
  - 目標おもちゃ
  - 必要な金額
  - 毎月買うもののリスト
  - 毎月の貯金額
  - 必要な期間（◯か月）

### 2.3 実行フェーズ

#### 2.3.1 月ごとの進行
ゲームは1か月ごとに進行します。4月からスタートし、5月、6月...と進みます。

**各月の流れ：**
1. 月の始まり：おこづかいをもらう（+500円など）
2. ハプニング発生（0〜1回/月）
3. お出かけイベント発生（2回/月・固定）
4. 月末：貯金額の確認

#### 2.3.2 お出かけイベント（誘惑）
お出かけの際、以下のような誘惑が発生します：

- 「おもちゃやさんで、このおもちゃがほしい！（◯◯円）」
- 「おかしやさんで、このおかしがほしい！（◯◯円）」
- 「ゲームセンターで、1かいだけあそびたい！（◯◯円）」

**プレイヤーの選択：**
- 「買う（遊ぶ）」→ お金が減る
- 「がまんする」→ 計画通り

**重要な仕組み：**
- 誘惑の内容と金額は、プレイヤーが立てた計画とは無関係にランダムで発生
- 計画で「おかしを買う」と決めていても、別の誘惑（おもちゃなど）が来ることもある
- これにより「計画通りにいかない現実」を体験する

#### 2.3.3 ハプニングイベント
月に0〜1回、以下のようなハプニングがランダムで発生します：

**プラスのハプニング：**
- 「おてつだいをして、おこづかいがもらえた！（+100円）」
- 「おじいちゃん・おばあちゃんからおこづかいをもらった！（+300円）」
- 「たんじょう日で、おかねをもらった！（+500円）」

**マイナスのハプニング：**
- 「おかねをおとしてしまった…（-200円）」

#### 2.3.4 進行状況の表示
画面には常に以下の情報が表示されます：
- 現在の月（例：4月、5月、12月、1月…）
- 現在の所持金
- 目標おもちゃと必要金額
- 計画との差（予定より多い/少ない）

### 2.4 結果フェーズ

#### 2.4.1 成功条件
計画期間内（または期間終了時）に、目標おもちゃの金額に到達する

#### 2.4.2 失敗条件
計画期間が終わっても、目標金額に到達しない

#### 2.4.3 フィードバック
**成功時：**
- 「よくできました！さくせんどおりにためられたね！」
- 目標おもちゃを手に入れた画像を表示
- 達成までの記録を表示（何か月かかったか、何回誘惑に勝ったか等）

**失敗時：**
- 「ざんねん…もうすこしだったね！」
- 不足金額を表示
- 「もっと小さいもくひょうにしてみる？」
- 「まい月、すこしだけがまんしてみる？」
- などのヒントを表示

#### 2.4.4 リトライ
- 「もういちどチャレンジ」ボタンで最初から何度でもやり直せる
- 前回の記録は保存されない（シンプルに）

### 2.5 イベントの拡張性
誘惑イベントは、将来的に追加・編集できる構造にします。
- イベントデータは配列やJSONで管理
- 新しいイベントを簡単に追加できる設計

## 3. 画面構成

### 3.1 画面一覧
1. タイトル画面
2. 説明画面（ルール説明）
3. 作戦立案画面
4. 実行画面（月ごとの進行）
5. 結果画面

### 3.2 各画面の詳細

#### 3.2.1 タイトル画面
**表示要素：**
- ゲームタイトル「おこづかいチャレンジ！」
- 「スタート」ボタン

**遷移：**
- 「スタート」→ 説明画面

---

#### 3.2.2 説明画面
**表示要素：**
- ゲームの目的の簡単な説明
  - 「ほしいおもちゃを手に入れるために、さくせんを立てよう！」
  - 「まい月おこづかいをもらえるよ」
  - 「お出かけの時に、買いたいものが出てくるよ」
  - 「さくせんどおりにためられるかな？」
- イラスト（貯金箱、おもちゃなど）

**操作：**
- 「さくせんを立てる」ボタン → 作戦立案画面

---

#### 3.2.3 作戦立案画面

**表示要素：**

1. **ステップ1: 目標おもちゃを選ぶ**
   - おもちゃのカード（画像、名前、価格）を複数表示
   - 選択したカードがハイライト表示される

2. **ステップ2: 毎月買うものを決める**
   - 「まい月、おこづかいは500円もらえるよ」と表示
   - 「まい月、何を買う？」
   - 選択肢は画像カード形式で表示：
     - 「なにも買わない」（貯金箱アイコン）
     - 様々なアイテムを画像カード表示（例：チョコ100円、アイス150円、グミ120円、ミニカー300円、ぬりえ400円、カード500円、UFOキャッチャー200円、レースゲーム100円、太鼓ゲーム300円など）
   - 各カードには必ず画像、名前、価格を表示
   - 画像が読み込めない場合は、画像枠内にアイテム名を表示
   - カードをクリック/タップで選択・追加
   - 複数選択可能（例：チョコとミニカーの両方を選ぶ）
   - 選択したアイテムはリスト表示され、削除も可能

3. **計画の確認エリア（リアルタイム更新）**
   - 「もくひょう：◯◯◯◯（◯◯◯◯円）」
   - 「まい月買うもの：」（選択したアイテムのリスト表示）
   - 「まい月のしはらい：◯◯◯円」（選択したアイテムの合計金額）
   - 「まい月たまるおかね：◯◯◯円」（500円 - 毎月の支払い）
   - 「もくひょうたっせいまで：◯か月ひつよう」（必要な月数を自動計算）
   - 目標金額に到達するかの判定結果（自動判定）
     - ✓「けいかくOK！◯か月でたっせいできるよ！」（緑色）
     - ✗「このけいかくだと◯か月かかるよ。もっとがまんする？」（黄色）
     - ✗「まい月のしはらいがおこづかいより多いよ！」（赤色）

**操作：**
- 「チャレンジする！」ボタン → 実行画面
  - ただし、目標金額に到達しない計画の場合は警告表示
  - 「このままだと失敗しちゃうよ。それでもチャレンジする？」
  - 「はい」「いいえ（なおす）」

---

#### 3.2.4 実行画面

**常時表示要素：**
- **ヘッダー部分：**
  - 現在の月：「◯月」（例：4月、5月、12月、1月…）
  - 所持金：「いまのおかね：◯◯◯円」
  - 貯金箱アイコン（所持金に応じて満たされる表現）

- **目標表示部分：**
  - 目標おもちゃの画像
  - 「もくひょう：◯◯◯◯円」
  - 進捗バー（目標までの達成度）

- **計画との比較：**
  - 「けいかくより◯◯円おおい/すくない」
  - ✓「けいかくどおり！」または ⚠「けいかくよりすくない…」

**イベント表示エリア：**
- 月初めメッセージ：
  - 「◯月になったよ！」
  - 「おこづかいをもらったよ！（+500円）」

- ハプニング発生時：
  - イラスト付きでメッセージ表示
  - 「おてつだいをしたよ！おこづかいがもらえた！（+100円）」
  - 「つぎへ」ボタン

- お出かけイベント発生時：
  - 「お出かけだよ！」
  - 誘惑アイテムを画像カードで表示
    - カードには必ず画像、名前、価格が表示される
    - 画像が読み込めない場合は、画像枠内にアイテム名を表示
    - 例：「チョコ（100円）」「ミニカー（300円）」「UFOキャッチャー（200円）」など
  - 選択ボタン：
    - 「買う（遊ぶ）」ボタン（大きく、目立つ）
    - 「がまんする」ボタン（少し控えめ）

  - 選択後のフィードバック：
    - 「買った」→ アイテム画像と「◯◯◯を買ったよ（-◯◯◯円）」
    - 「がまんした」→「よくがまんできたね！」
    - 所持金が更新される

- 月末メッセージ：
  - 「◯月おわり！」
  - 「こんげつのちょきん：◯◯◯円」
  - 「つぎの月へ」ボタン

**操作：**
- 各イベントで選択 → 次のイベントまたは次の月へ
- 計画期間終了 → 結果画面

---

#### 3.2.5 結果画面

**成功時：**
- 「せいこう！」の大きな文字
- 目標おもちゃの画像（キラキラエフェクト）
- 「よくできました！◯◯◯◯を手に入れたよ！」
- 達成記録：
  - 「◯か月でたっせい！」
  - 「ゆうわくに◯回かった！」
- 「もういちどあそぶ」ボタン

**失敗時：**
- 「ざんねん…」の文字
- 「もうすこしだったね！」
- 不足金額の表示：
  - 「もくひょう：◯◯◯◯円」
  - 「いまのおかね：◯◯◯円」
  - 「あと◯◯◯円たりなかった！」
- フィードバックメッセージ（ランダムで表示）：
  - 「もくひょうが大きすぎたかな？もっと小さいもくひょうにしてみよう」

  - 「けいかくをもっと長くしてみるのもいいよ」
- 「もういちどチャレンジ」ボタン → タイトル画面

## 4. 操作方法

### 4.1 基本操作
- **すべての操作はボタンのクリック/タップで完結**
- マウス、タッチパネル両対応
- テキスト入力は一切なし

### 4.2 ボタン設計
- **大きく、わかりやすいボタン**
  - タップしやすいサイズ（最低44x44px以上）
  - ボタンの文字は大きく、読みやすく

- **色による識別**
  - 「はい」「買う」「進む」系：明るい色（青、緑など）
  - 「いいえ」「がまんする」系：落ち着いた色（グレーなど）
  - 重要なボタン：大きく、目立つ色

- **ホバー/タップ時のフィードバック**
  - ボタンが反応していることを視覚的に示す
  - 色の変化、拡大など

### 4.3 文字表記ルール

#### 4.3.1 使用する漢字
**小学1〜2年生で習う漢字のみ使用**
- 1年生：一、二、三、円、月、日、出、入、手、目 など
- 2年生：今、計、買、遊、友 など

#### 4.3.2 振り仮名
- **すべての漢字に振り仮名を付ける**
- ルビの形式：`<ruby>漢字<rt>かんじ</rt></ruby>`

#### 4.3.3 表記の工夫
- できるだけひらがな表記を優先
- 「おこづかい」「おもちゃ」「おかし」などはひらがな
- 数字はアラビア数字を使用（500円、3か月など）

#### 4.3.4 文章の長さ
- 短く、わかりやすい文章
- 1文は1行程度に収める
- 難しい言葉を避ける

## 5. ゲームバランス

### 5.1 基本パラメータ

| 項目 | 値 | 備考 |
|------|------|------|
| 月のおこづかい | 500円 | 固定値 |
| 最小目標金額 | 1,500円 | 3か月で達成可能 |
| 最大目標金額 | 6,000円 | 12か月で達成可能 |
| 計画最大月数 | 12か月 | 1年間 |
| お出かけ頻度 | 2回/月 | 固定 |
| ハプニング頻度 | 0〜1回/月 | ランダム |

### 5.2 誘惑の金額設定

購入可能なアイテムの価格帯：
- 100〜200円：小さな誘惑（例：お菓子類）
- 100〜300円：中程度の誘惑（例：ゲームセンター）
- 300〜800円：大きな誘惑（例：おもちゃ類）

### 5.3 ハプニングの影響

| ハプニング | 影響 | 発生確率 |
|-----------|------|---------|
| おてつだい | +100円 | 低 |
| おじいちゃんから | +300円 | 低 |
| たんじょう日 | +500円 | 非常に低（1年に1回程度） |
| お金をおとす | -200円 | 非常に低 |

### 5.4 難易度バランス

**初心者向けの目標：**
- 目標金額：1,500〜2,000円
- 計画期間：3〜4か月
- 誘惑に1〜2回負けても達成可能

**中級者向けの目標：**
- 目標金額：3,000〜4,000円
- 計画期間：6〜8か月
- 誘惑に1回以下しか負けられない

**上級者向けの目標：**
- 目標金額：5,000〜6,000円
- 計画期間：10〜12か月
- 誘惑にほとんど負けられない

### 5.5 プレイ時間
- 1プレイ：**5〜10分**
- 1か月あたり：30秒〜1分
- 3か月プレイ：約3分
- 6か月プレイ：約6分
- 12か月プレイ：約10分

## 7. 技術仕様（概要）

### 7.1 使用技術
- HTML5
- CSS3（レスポンシブデザイン）
- JavaScript（Vanilla JS または軽量フレームワーク）

### 7.2 データ構造
- **ゲーム状態：** JavaScriptオブジェクトで管理
- **アイテムデータ：** JSON形式で配列管理（拡張可能）
  - 各アイテムには、id、name、price、imageを持つ
  - 例：`{id: "item_1", name: "チョコレート", price: 100, image: "assets/images/items/chocolate.png"}`
- **目標おもちゃデータ：** JSON配列で管理
- **誘惑イベントデータ：** アイテムIDを参照する形式で管理
- **保存機能：** なし（シンプルに、毎回新規プレイ）

### 7.3 拡張性
- アイテムデータを外部JSONファイルとして管理可能
- 新しいアイテムを簡単に追加できる構造
  - JSONファイルに新しいエントリを追加するだけ

- 目標おもちゃも配列で管理し、追加・変更が容易
- すべてのアイテムは統一的に管理され、UI表示も自動生成

## 9. まとめ

このゲームは、小学1年生が「数字を使って計画を立てる力」を楽しく学べる教育的シミュレーションゲームです。

**コア体験：**
1. 目標を決める
2. 計画を立てる
3. 誘惑に直面する
4. 結果から学ぶ

**教育的価値：**
- 無理な計画は実行できないことを体験する
- 実現可能な計画とは何かを考える
- 計画の大きさと実効性のバランスを学ぶ

**ゲームとしての楽しさ：**
- シンプルでわかりやすい操作
- 明るく楽しいビジュアル
- 何度でもやり直せる安心感
- 達成感のあるフィードバック
